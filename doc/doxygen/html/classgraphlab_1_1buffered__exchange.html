Detailed Description

template<typename T>
class graphlab::buffered_exchange< T >

The buffered exchange provides high performance exchange of bulk data between machines. The basic usage is simple:

For instance, if we are doing bulk exchanges of integers:

buffered_exchange<int> exchange(dc, numthreads);
.. In parallel .. {
exchange.send([target machine], [value to send to target], [thread id])
exchange.partial_flush([thread id]);
}
.. now in 1 thread ..
exchange.flush()
.. In parallel .. {
procid_t proc;
buffered_exchange<int>::buffer_type buffer; // (an array of integers)
while(exchange.recv(proc, buffer)) {
process array of integers (buffeer) which were sent by proc
}
}
Note
The buffered exchange sends data in the background, so recv can be called even before the flush calls.
See also
graphlab::fiber_buffered_exchange

Constructs a buffered exchange object.

dc The master distributed_control object num_threads The number of threads to support. This is essentially the number of fine-grained locks to use. This does not need to match the total number of threads used during the exchange process, but there are performance / contention advantages if this matches. max_buffer_size The size of the per thread and per target send buffer.

Returns a collection of T sent by ret_proc.

Parameters
ret_procOn successful return, will contain a valid procid indicating that the values in the buffer were sent by that process.
ret_bufferA sequence of values sent by ret_proc
try_lockIf true, will not acquire the lock if the lock is contended. Useful for polling the receive buffer while sending is occuring.
Returns
True on success and there are values in the buffer. False if the receive buffer is empty. Or if try_lock is set, False may also indicate the buffer lock is being contended.
